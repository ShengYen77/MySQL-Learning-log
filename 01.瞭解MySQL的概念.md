# ⭐MySQL

是一種**關聯式資料庫管理系統(RDBMS)**，簡單來說，它是一個用來儲存、管理、查詢資料的工具。

可以說它就像是一個超大型的Excel，但不同的是：

Excel 表格用滑鼠操作，而 MySQL 用 **SQL 指令語言** 操作。

它更強大，可以讓成千上萬的人同時存取資料，能處理很多複雜的資料邏輯與查詢。

# MySQL基本概念

| 名稱            | 說明                                    |
| ------------- | ------------------------------------- |
| **資料庫（Database）** | 儲存不同資料的容器，例如「顧客資料庫」、「訂單資料庫」           |
| **資料表（Table）**    | 資料庫裡的一張表格，例如顧客表、訂單表，每一張表都會有「欄位」和「資料列」 |
| **SQL（語言）**       | Structured Query Language，一種用來操作資料的語言 |
| **資料列（Row）**      | 表中的一筆資料，例如某位顧客的姓名、電話、地址等      |
| **資料欄（Column）**   | 表中的一種資料欄位，例如「姓名」、「電話」等欄         |

接著我們來比較**Oracle Database** 、 **MySQL** 、 **PostgreSQL** 三者之間的差異

| 項目    | **Oracle Database** | **MySQL**                 | **PostgreSQL**                    |
| ----- | ------------------- | ------------------------- | --------------------------------- |

# 關係型資料庫(Relational Database)

是一種以「表格（Table）」為基礎，資料之間透過「關聯（Relationship）」來管理的資料庫系統。

資料以「列（row）」與「欄（column）」儲存，每一筆資料稱為一筆紀錄(record)，每一個欄位定義一種資料屬性(attribute)。

「關係（Relation）」是數學中的集合概念，資料表之間的「關聯」透過：

**主鍵（Primary Key）**

**外鍵（Foreign Key）**

來建立表格間的關係，例如員工屬於部門(employee → department)、客戶下訂單(customer → order)

# 關係型資料庫的特性

| 特性           | 說明                                    |
| ------------ | ------------------------------------- |
| 結構化          | 資料以表格儲存，格式一致                          |
| 支援 SQL 語法    | 使用標準 SQL 指令來查詢與操作                     |
| 支援資料完整性      | 主鍵、外鍵、NOT NULL、UNIQUE 等約束確保資料正確性      |
| 支援交易機制（ACID） | 保障交易的**原子性、一致性、隔離性、持久性**，適合金融、企業等關鍵應用 |
| 支援資料關聯查詢     | 可以用 `JOIN` 查出多張表格的相關資料                |

# 跟「非關係型資料庫」的差異是什麼？

| 比較項目  | 關係型資料庫（RDB）      | 非關係型資料庫（NoSQL）   |
| ----- | ---------------- | ---------------- |
| 結構    | 表格結構，固定欄位        | 文件、鍵值、圖形、文件等多樣形式 |
| 擴充性   | 垂直擴充（scale-up）   | 水平擴充（scale-out）  |
| 一致性強度 | 強一致性（ACID）       | 最終一致性（BASE）      |
| 使用情境  | 銀行、ERP、CRM、報表系統等 | 即時訊息、社群、IOT、大數據  |

📌筆記 : 

**關係型資料庫是一種以表格為基礎的資料庫系統，透過主鍵與外鍵建立資料關聯，支援 SQL 查詢、交易機制（ACID）與資料完整性，適用於高度結構化且資料關聯明確的場景。常見的系統有 MySQL、PostgreSQL、Oracle 等。**
